# Обзор проекта

Данный проект представляет собой высокопроизводительный движок для 2D-игры (например, роуглайка, ARPG или шутера с видом сверху), написанный на TypeScript и использующий современные подходы к веб-разработке и геймдеву.

## Технологический стек

- **Графика (Рендеринг):** [Pixi.js](https://pixijs.com/) — используется для быстрого 2D-рендеринга через WebGL.
- **Архитектура игровой логики:** [bitECS](https://github.com/NateTheGreatt/bitECS) — эффективная реализация Entity Component System с использованием Data-Oriented Design (DOD). Обеспечивает высокую производительность за счет кэширования массивов (TypedArrays).
- **Физика:** [Matter.js](https://brm.io/matter-js/) — 2D-физический движок для просчета столкновений (Rigid Body Collision) и перемещений.
- **Генерация мира:** Используются алгоритмы BSP (Binary Space Partitioning), дополненные библиотеками `rot-js`, `simplex-noise` и `seedrandom` для процедурной генерации подземелий/уровней.
- **Пользовательский интерфейс (UI):** React + Zustand + TailwindCSS + Motion + Localization System. React накладывается поверх игрового холста (Canvas) и используется строго для HUD и меню инвентаря/настроек, тогда как Pixi отвечает за саму игру. Локализация реализована через `src/localization/`.
- **Событийная шина:** Кастомный `EventBus` для общения между слабо связанными частями системы.
- **Управление памятью:** `ObjectPool` для переиспользования частых объектов (снаряды, эффекты, враги), что минимизирует вызовы сборщика мусора (Garbage Collector).

## Основные возможности движка

1. **Строгое разделение ответственности (Separation of Concerns).** Логика, визуализация и физика работают независимо друг от друга через системы.
2. **Процедурная генерация карт.** Наличие `BSPMapGenerator` и `MapGenerator` позволяет на лету создавать рандомизированные уровни.
3. **Производительность.** Синергия bitECS и ObjectPool позволяет выводить большие скопления врагов без просадки фреймрейта.
4. **Гибридный рендеринг.** Комбинирование canvas-подхода (Pixi.js для геймплея) и DOM-подхода (React для UI) дает максимальную гибкость.
5. **Система локализации и Главное меню.** Удобная интеграция текстовых ресурсов и начальный экран для управления состоянием игры.

## Структура проекта

- `src/config/` — Конфигурационные файлы (Настройки графики, физики, общие константы).
- `src/core/` — Ядро игрового движка, классы инициализации и Главный цикл (`GameLoop`).
- `src/ecs/` — Компоненты, Фабрики сущностей и Системы.
- `src/models/` — Интерфейсы и общие TypeScript типы.
- `src/services/` — Глобальные сервисы (например, Обертка для физики, Пулы объектов).
- `src/stores/` — Стейт-менеджер (Zustand) для связи игровой логики с React UI.
- `src/utils/` — Вспомогательные утилиты, включая продвинутые инструменты генерации уровней.
- `src/viewmodels/` — ViewModel для трансформации ввода пользователя в игровые события.
- `src/views/` — Классы и логика отображения, управление WebGL/Pixi поверхностями.
- `src/components/` — React-компоненты пользовательского интерфейса (HUD).
